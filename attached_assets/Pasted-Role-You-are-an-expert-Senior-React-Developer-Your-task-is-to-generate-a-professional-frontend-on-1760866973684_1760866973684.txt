Role: You are an expert Senior React Developer. Your task is to generate a professional, frontend-only website application for a law firm.

Core Technologies:

Framework: React.js (using Hooks)

Bundler/Environment: Vite + React

Styling: Tailwind CSS

Routing: react-router-dom

Internationalization (i18n): i18next and react-i18next

Project Requirements:

Project Structure: Create a clean, component-based structure. Provide the code for each file separately, clearly labeling the file path (e.g., src/App.jsx, src/pages/HomePage.jsx).

Tailwind CSS Setup: Provide tailwind.config.js and src/index.css configured for a Vite + React project. The design should be professional, clean, and trustworthy (e.g., using a color palette of navy blue, white, and gold/beige).

Routing: Set up react-router-dom for all pages. The routes should include /, /about, /practice-areas, /practice-areas/:slug, /team, /team/:lawyerId, and /contact.

Multi-Language (i18n):

The site must support English (EN), Turkish (TR), and Arabic (AR).

Use the i18next library. Create a config file src/i18n.js.

Crucially, pre-fill the translation JSON files with relevant, professional text for a law firm, as specified in section 8.

All text in components must be rendered using the useTranslation() hook (e.g., t('nav.about')).

RTL Support:

The application must dynamically set dir="rtl" on the <html> tag when Arabic is selected. This logic should be in App.jsx using a useEffect hook that listens to i18n.language.

All Tailwind CSS classes must use logical properties (e.g., ms-4 instead of ml-4, pe-2 instead of pr-2, text-start instead of text-left) to ensure the layout flips correctly for RTL.

Mock Data: Create a src/mockData.js file to export arrays for lawyers and practiceAreas. This data will be imported and used by components.

File Structure & Code to Generate:

1. Config: tailwind.config.js (Standard Vite/React setup)

2. Config: src/index.css (With @tailwind base;, @tailwind components;, @tailwind utilities;)

3. i18n Config: src/i18n.js (Configuration for i18next, react-i18next, and HttpBackend to load JSON files from /locales/)

4. Main App: src/main.jsx (Import and initialize i18n.js here)

5. Main App: src/App.jsx (Main layout with Navbar, Footer, and <Outlet />. Include the useEffect hook for RTL support.)

6. Router: src/Router.jsx (Define all routes using createBrowserRouter: /, /about, /practice-areas, /practice-areas/:slug, /team, /team/:lawyerId, /contact.)

7. Mock Data: src/mockData.js

Create a practiceAreas array. Each object needs: id, slug, icon (e.g., a string for a placeholder icon), titleKey, descriptionKey, fullDetailsKey.

Create a lawyers array. Each object needs: id, name, imageUrl, titleKey (e.g., 'partner', 'associate'), bioKey, educationKey.

9. Component Files (src/components/):

src/components/Navbar.jsx:

Uses <Link> for navigation links (Home, About, Practice Areas, Team, Contact).

Includes <LanguageSwitcher />.

Responsive "hamburger" menu.

src/components/LanguageSwitcher.jsx:

Dropdown to select 'en', 'tr', or 'ar', calling i18n.changeLanguage().

src/components/Footer.jsx:

Professional footer with links, contact info, and copyright (all text from t()).

src/components/PracticeAreaCard.jsx:

Takes an area prop (from mockData.js).

Renders an icon, t(area.titleKey), and t(area.descriptionKey).

Wraps the card in a <Link> to /practice-areas/${area.slug}.

src/components/LawyerProfileCard.jsx:

Takes a lawyer prop.

Renders lawyer.imageUrl, lawyer.name, and t(lawyer.titleKey).

Wraps the card in a <Link> to /team/${lawyer.id}.

10. Page Files (src/pages/):

src/pages/HomePage.jsx:

Hero Section: Full-width with a professional background image, t('hero.title'), t('hero.subtitle'), and a CTA button t('hero.cta').

Practice Areas Section: Title t('home.practiceAreasTitle') and a grid of 3-4 <PracticeAreaCard /> components (slicing from mockData.js).

Team Section: Title t('home.teamTitle') and a grid of 3 <LawyerProfileCard /> components.

src/pages/PracticeAreasPage.jsx:

Renders a full grid of all <PracticeAreaCard /> components from mockData.js.

src/pages/PracticeAreaDetailPage.jsx:

Uses useParams() to get the :slug.

Finds the matching area from mockData.js.

Displays t(area.titleKey) and the full t(area.fullDetailsKey).

src/pages/TeamPage.jsx:

Renders a full grid of all <LawyerProfileCard /> components.

src/pages/LawyerProfilePage.jsx:

Uses useParams() to get the :lawyerId.

Finds the matching lawyer from mockData.js.

Displays lawyer.imageUrl, lawyer.name, t(lawyer.titleKey), t(lawyer.bioKey), and t(lawyer.educationKey).

src/pages/ContactPage.jsx:

Displays a contact form (t('contactForm.title'), etc.) and placeholder map.
